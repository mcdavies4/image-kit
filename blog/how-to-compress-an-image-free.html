What quality setting should I use?

  • 90–100% — near-lossless, minimal size reduction. Best for print or archiving.
  • 75–85% — the sweet spot for web images. Sharp enough to look great, small enough to load fast.
  • 60–74% — noticeable compression but still acceptable for thumbnails and previews.
  • Below 60% — heavy compression, artefacts may appear. Use only when file size is critical.

Why compress images locally?

Tools like TinyPNG, Squoosh and Compressor.io upload your images to their servers before processing. For personal photos, business graphics or confidential documents that include images — that is an unnecessary privacy risk.

RightImageKit uses the browser's built-in Canvas API to compress images entirely on your device. There is no server involved at any point. You can even disconnect from the internet after loading the page and it will still work.

Common uses for image compression

  • Website optimisation — smaller images mean faster page load times and better SEO
  • Email attachments — most email clients struggle with images over 5–10MB
  • Social media uploads — Instagram, X and LinkedIn recompress images anyway — pre-compress to control the output
  • WhatsApp and messaging — avoid the quality loss from in-app compression by compressing first
  • Storage saving — reduce the size of photo archives on your phone or hard drive

Frequently asked questions

Can I compress multiple images at once?

RightImageKit's free Compress tool handles one image at a time. The Pro Bulk Compress tool lets you compress multiple images in one go and download them as a ZIP.

Can I compress a HEIC photo from my iPhone?

Yes — use the HEIC to JPG tool first to convert it, then compress the resulting JPG. Both tools are free.

Does compressing change the image dimensions?

No. Compression only reduces the file size by reducing image data — the pixel dimensions stay exactly the same. Use the Resize tool if you need to change dimensions.

What is the difference between lossy and lossless compression?

Lossy compression (JPG, WebP) permanently removes some image data to reduce file size. Lossless compression (PNG with optimisation) reduces size without removing any data. RightImageKit's quality slider lets you control how much lossy compression to apply.
